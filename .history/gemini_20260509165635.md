3. ОГРАНИЧЕНИЯ ИНСТРУМЕНТОВ (TOOLING CONSTRAINTS)

Критически важное правило по работе с файловой системой:

4. АРХИТЕКТУРА И ПРОЕКТИРОВАНИЕ (COMPOSE DESKTOP MVI)

Весь код должен соответствовать строгому разделению слоев. Смешивание логики UI, I/O и математики недопустимо.

Backend/Core (Java 21+)

Domain Layer (Предметная область):

Содержит бизнес-сущности (FileNode, DirectoryTree), абстрактные алгоритмы (Squarified Treemap).

НИКАКИХ зависимостей от Compose, UI или конкретной реализации файловой системы.

I/O & Infrastructure Layer:

Реализация сканирования диска (New I/O - java.nio.file).

Управление пулами потоков (ForkJoinPool или Virtual Threads).

State/ViewModel Layer (Оркестрация):

Связующее звено. Инициирует сканирование, агрегирует данные и подготавливает строго иммутабельный стейт (UI Model) для передачи в слой представления.

Frontend/UI (Compose Multiplatform)

Presentation Layer:

Декларативный UI. Подписывается на State/Flow из ViewModel.

Canvas Rendering:

Главный экран отрисовки дерева файлов — это "глупый" компонент, использующий Canvas (drawRect). Он получает готовый массив прямоугольников (с предвычисленными координатами и цветами) и только рисует их.

5. КАЧЕСТВО КОДА (CODE QUALITY)

NO HARDCODE: Никаких захардкоженных строк, путей или системных параметров.

DRY & SOLID: Классы имеют одну ответственность. Алгоритм Treemap отделен от сканера диска, сканер диска отделен от рендера.

Иммутабельность: Критически важно! Данные, передаваемые между фоновым I/O потоком и UI потоком рендеринга, должны быть 100% иммутабельными (Record classes), чтобы избежать ConcurrentModificationException.

Типизация: Максимально строгая типизация. Широкое использование record из Java.

Размер: Файлы не должны быть раздутыми. Развернутые ответы при генерации кода обязательны.

6. ДОПОЛНИТЕЛЬНЫЕ ИНСТРУКЦИИ ДЛЯ JSpaceStat (WINDIRSTAT CLONE)

I/O Ограничения: Для обхода файловой системы использовать ТОЛЬКО java.nio.file (NIO). Устаревший java.io.File под строгим запретом из-за проблем с производительностью.

Ошибки ОС: Обход диска не должен падать. Код обязан молча игнорировать (перехватывать) AccessDeniedException, зацикливания симлинков и защищенные системные файлы.

Многопоточность: UI-поток (Main Thread) блокировать ЗАПРЕЩЕНО. Весь I/O и расчет математики для Treemap выносится в фоновые потоки.

Производительность рендера: Строго запрещено создавать отдельные Compose-элементы (ноды DOM/UI-дерева) для каждого файла на диске. Отрисовка десятков тысяч узлов выполняется только через единый вызов на Canvas API.
